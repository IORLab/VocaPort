# VocaPort 贡献者许可协议（CLA）

本协议适用于你向 `IORLab/VocaPort` 提交并希望被项目接收的代码、文档、测试、脚本、设计资产和其他受版权保护的材料。

## 1. 定义

- “项目”指 `VocaPort` 及其代码仓库 `IORLab/VocaPort`。
- “项目维护方”指当前仓库所有者，以及未来实际控制该项目并承接其权利义务的实体或个人。
- “贡献”指你通过 pull request、patch、commit、issue 附件、邮件或其他方式提交给项目并希望被项目接收的材料。

## 2. 版权归属

除非你另行书面说明，你仍然保留自己对贡献的版权。

## 3. 你授予项目维护方的权利

你授予项目维护方一项全球范围、长期有效、不可撤销、非独占、免版税、可再授权的许可，使项目维护方可以：

- 使用、复制、修改、改编、合并、公开展示、公开表演和分发你的贡献；
- 将你的贡献作为 `VocaPort` 的一部分使用，或单独使用；
- 以当前的 `AGPL-3.0-only`、未来版本的开源协议、源代码可见协议、双重许可、商业许可或专有许可重新发布或再授权你的贡献。

## 4. 你的保证

你确认：

- 你有合法权利提交该贡献并授予本协议中的权利；
- 如果贡献包含第三方材料，你已经获得足够的许可，且这些材料不会与你在本协议中授予的权利冲突；
- 你提交贡献的行为不会违反你与雇主、学校、客户或其他第三方之间的保密、竞业或知识产权约束。

## 5. 当前项目协议

当前项目代码仓库按 `AGPL-3.0-only` 发布；本协议不会削弱外部接收者根据该开源协议已经获得的权利。本协议的目的是让项目在未来需要双授权、商用授权或调整仓库协议时，仍然具备再授权空间。

## 6. 无担保

除非你与项目维护方另有书面约定，你的贡献按“现状”提供，不附带任何明示或默示担保。

## 7. 接受方式

在仓库引入自动化 CLA 工具前：

- 你需要在 pull request 模板中勾选 CLA 确认项，或在 PR 描述中明确写明你同意本协议；
- 项目维护方仅在收到该确认后合并贡献。

如果你不同意本协议，请不要向 `VocaPort` 提交贡献。
